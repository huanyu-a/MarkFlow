# 使用微型 HTML 片段渲染风格预览缩略图

_来源：eda6dc8 → 976215b 提交周期内记录的编码计划——内容为规划时意图，实现可能滞后或有出入。_

**状态：** accepted

## 背景
指令库（PromptLibrary）需要展示多种设计风格的视觉预览，以便用户快速选择。需要在保证视觉保真度和降低维护成本之间做出权衡。

## 决策驱动
- 零网络请求依赖
- 极小的 Bundle 体积
- 避免构建时截图工具链
- 真实反映风格特征（字体、排版、配色）

## 备选方案
- **预生成 WebP/JPG 截图作为缩略图** _（已否决）_ — 优点：视觉效果最准确；缺点：需要构建时工具链自动生成截图；增加静态资源体积；维护成本高（风格变更需重新截图）
- **纯 CSS/SVG 抽象骨架模拟** _（已否决）_ — 优点：体积极小，无外部依赖；缺点：难以真实反映特定风格的字体族、间距密度等细节，辨识度低
- **渲染内联微型 HTML 片段 (previewHtml)** — 优点：真实呈现字体、排版、配色等核心特征；零网络请求；无需外部图片资源；可通过 CSS transform 缩放适配容器；缺点：需要为每个风格编写约 15 行的微型 HTML/CSS 代码

## 决策
在 `DesignStyle` 接口中新增 `previewHtml` 字段，创建 `src/modes/html/StyleThumbnail.tsx` 组件。该组件渲染风格自带的微型 HTML 片段，并通过 CSS `transform: scale()` 在固定尺寸容器中展示。若缺失 `previewHtml` 则降级为抽象骨架。

## 影响
提升了风格选择的直观性和准确性，且无需维护庞大的图片资源库。但增加了 `designPrompts.ts` 的数据维护工作量，需为每个内置风格编写对应的微型 HTML 片段。
